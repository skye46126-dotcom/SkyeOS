# LifeOS AI 方向执行计划 v1

## 1. 目标

- AI增强：接入 VCP 中间层进行自然语言结构化。
- 保底可用：VCP 不可用时自动降级本地规则解析。
- 先预览后提交：解析结果必须人工确认后入库。

## 2. 架构原则

1. 双通道
- VCP 通道：高质量解析与建议
- Rule 通道：规则脚本保底

2. 统一抽象
- `ParserEngine` 统一接口，屏蔽底层差异

3. 稳定优先
- 任何 AI 故障不影响日常录入

## 3. 模式定义

- `auto`：默认，优先 VCP，失败自动降级 Rule
- `vcp`：仅 VCP
- `rule`：仅 Rule

## 4. 核心组件

1. `ParserMode`（枚举）
- `AUTO/VCP/RULE`

2. `ParseDraft`
- `kind`: time_log | income | expense | learning | project
- `payload`: 结构化字段
- `confidence`: 0~1
- `source`: vcp | rule
- `warnings`: 告警列表

3. `ParserEngine` 接口
- `ParseResult parse(String rawText, String dateHint)`

4. `RuleParserEngine`
- 正则 + 关键词解析
- 未识别项写入 `unknown_items`

5. `VcpParserEngine`
- 调 SkyOS backend 的 AI 网关
- backend 再调用 VCP

6. `AiParseOrchestrator`
- 按模式编排
- `auto` 中处理降级与错误兜底

## 5. 接口设计（Backend）

- `POST /api/v1/ai/parse`
  - 入参：`raw_text`, `context_date`, `parser_hint`
  - 出参：`request_id`, `draft_items`, `confidence`, `warnings`

- `POST /api/v1/entries/commit`
  - 入参：`request_id`, `entries`
  - 要求：幂等 + 事务提交

## 6. UI 流程

1. 输入自然语言
2. 点击 AI 解析
3. 展示草稿（可编辑）
4. 一键确认入库
5. 返回成功/失败明细

## 7. 保底策略

- VCP 超时/5xx/网络异常 -> 自动切 Rule
- Rule 失败项进待补队列
- 显示提示：“已切换保底模式”

## 8. 测试计划

- 解析正确性测试（时间/收入/支出/学习）
- 降级路径测试（VCP失败自动切 Rule）
- 幂等测试（同 request_id 不重复写入）
- 回归测试（手动录入不受影响）

## 9. 执行里程碑

- M1：抽象层与数据结构
- M2：Rule 解析引擎
- M3：VCP 引擎接入
- M4：AI 预览确认页
- M5：测试与生产安全收口
