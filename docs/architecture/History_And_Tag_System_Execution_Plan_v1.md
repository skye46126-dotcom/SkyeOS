# LifeOS 历史账本与标签系统执行计划 v1（PRD 对齐版）

## 1. 背景与目标

当前系统已具备录入与基础复盘能力，但“历史回看”和“标签可视化”仍不足，导致以下问题：

- 收入/支出/时间/学习历史不易追踪。
- 无法按日/周/月快速筛选并复盘趋势。
- 标签数据已有底层表，但缺少管理页、选择器和看板。

本计划目标：

- 建立完整历史账本视图（先收入，再扩展全类型）。
- 建立可维护的标签系统（标签库 + 录入 + 看板）。
- 保持现有数据结构稳定，分阶段落地，不中断当前使用。

## 2. PRD 对齐范围（v1）

### 2.1 历史账本（优先）

- 收入历史列表页（按日/周/月筛选）。
- 收入趋势摘要（本期条数、本期金额、来源分布）。
- 来源/类型汇总卡片。
- 项目收入排行（先基础版，后增强）。
- 历史查看必须覆盖：收入、支出、时间、学习（分阶段完成）。

### 2.2 标签系统（优先）

- 标签库管理页：新增、分组、启停。
- 录入页多标签选择器（时间/学习/项目优先）。
- 标签看板：Top 标签、标签时间占比、标签支出占比。
- 标签体系要求：
  - 时间一级分类固定：生活/学习/工作/娱乐/其他。
  - 二级标签支持自定义与 emoji。
  - 项目、投资、支出、学习均可打标签（分阶段落地）。

### 2.3 成本与价值（并行推进）

- 理想时薪、实际时薪、时间负债。
- 项目时间成本、项目现金成本、项目利润与盈亏平衡线。
- 历史趋势需支持按日/周/月复盘。

## 3. 分阶段实施

### M1（本周）：历史收入闭环

交付：

- 复盘页新增“收入历史”区块。
- 支持按日/周/月窗口过滤。
- 展示最近 N 条收入流水 + 小计金额。
- 对齐复盘页日/周/月窗口。

验收：

- 任一窗口可看到对应历史收入。
- 小计金额与明细可核对。
- 可回答“过去这段时间我靠什么来源赚的钱”。

---

### M2（下周）：通用历史账本

交付：

- 新增“历史账本”页（收入/支出/时间/学习统一入口）。
- 支持类型筛选 + 关键词搜索 + 时间窗口筛选。
- 增加导出（CSV）预留接口。

验收：

- 用户 30 秒内可定位到任一历史记录。
- 历史记录与总览数字一致。

---

### M3（下周）：标签库管理

交付：

- 新增标签管理页：
  - 创建标签（名称、emoji、分组、作用域）
  - 启停标签
  - 系统标签只读，自定义可编辑
- 标签作用域：
  - time / project / income / expense / learning / global

验收：

- 可新增并在录入页选择新标签。
- 停用标签后不再出现在选择器里。

---

### M4（两周内）：标签看板与关联分析

交付：

- 复盘页新增标签看板：
  - Top 标签（时间）
  - Top 标签（支出）
  - 标签-项目贡献关系（基础版）
- 提供标签维度钻取入口（点标签看明细）。

验收：

- 用户能回答“时间花在了哪些标签上”。
- 用户能回答“哪些消费标签占比过高”。

---

### M5（两周内）：成本模型与时间负债看板

交付：

- 首页/复盘新增理想时薪、实际时薪、时间负债。
- 项目详情新增：项目时间成本、现金成本、总成本、项目利润。
- 增加盈亏平衡线展示：`时间投入 × 时间成本`。

验收：

- 可回答“这个项目是赚还是亏”。
- 可回答“我当前是在减少还是扩大时间负债”。

## 4. 数据与接口改造要点

### 4.1 历史账本

- 优先复用 `recent records` 数据流。
- M2 新增专用查询接口（按类型/日期窗口/分页）。
- 金额统一 cents 存储，展示时按货币格式化。

### 4.2 标签系统

- 复用现有 `tag` 表与关联表。
- 增量补充：
  - tag.scope
  - tag.emoji
  - tag.sort_order
- 新增关联表（若缺失）：
  - project_tag / income_tag / expense_tag

## 5. UI 与交互原则

- 历史优先“可检索、可筛选、可核对”。
- 标签优先“易选择、可理解、可统计”。
- 所有看板都提供“查看明细”的下一跳。

## 6. 风险与控制

- 风险：历史数据量增长导致卡顿。
  - 控制：分页 + 索引 + 后台线程查询。
- 风险：标签过多导致录入复杂。
  - 控制：按作用域分组 + 常用标签置顶 + 停用机制。
- 风险：统计口径不一致。
  - 控制：统一窗口定义（日/周/月）与金额单位（cents）。
