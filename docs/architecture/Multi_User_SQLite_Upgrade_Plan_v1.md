# SkyeOS 多用户升级计划（保留 SQLite）v1

## 1. 目标与边界

- 保留现有 SQLite，不切换 PostgreSQL。
- 从单用户升级到 3-4 人可用。
- 手动录入保留为主路径，AI/VCP 作为增强能力。
- 优先稳定、可回滚、可维护，不做重型架构。

## 2. 总体策略

- 数据主线仍是“本地记录 + 云端同步/备份”。
- 多用户改造核心是：身份认证 + 数据隔离 + 并发稳定。
- 协作先只做“项目共享”，其余数据默认私有。

## 3. 阶段计划

### M0（1-2 天）基线冻结与备份

任务：
- 打版本基线 Tag（例如：`v0.1-single-user-stable`）。
- 增加一键备份脚本：数据库文件、备份目录、部署配置。
- 固化接口清单与数据表清单，作为迁移对照。

验收：
- 能在 30 分钟内回滚到基线版本。
- 备份恢复后，`/health` 和关键接口可用。

---

### M1（3-5 天）账号体系与数据隔离（必须先做）

任务：
- 新增 `users`、`sessions`（或 token）表。
- 登录接口（最小版）：`login/logout/me`。
- 业务主表增加 `owner_user_id` 字段，并建立索引。
- 所有读写接口强制注入用户上下文并按用户过滤。
- 管理策略：不开放注册，管理员手动创建账号。

验收：
- 用户 A 无法读取/修改用户 B 数据。
- 同一设备切换账号后数据视图正确隔离。

---

### M2（3-4 天）SQLite 并发稳定化

任务：
- 启用与校验 SQLite 参数：
  - `PRAGMA journal_mode=WAL`
  - `PRAGMA busy_timeout=5000`
  - `PRAGMA synchronous=NORMAL`
- 写操作改为短事务，禁止长事务包裹网络调用。
- 备份/恢复/批处理加互斥锁，避免重任务并发。
- 增加锁等待与慢查询日志。

验收：
- 3-4 人常规并发写入不出现持续性失败。
- `database is locked` 发生率可控且可自动重试。

---

### M3（4-6 天）轻量协作（仅项目共享）

任务：
- 新增 `project_member(project_id, user_id, role, created_at)`。
- 权限角色先做两档：
  - `owner`：项目管理、成员管理
  - `member`：项目记录读写
- 项目相关记录（时间/收入/学习）可挂到共享项目。
- 个人支出、个人日记、个人设置默认私有不共享。

验收：
- 项目成员可协作，非成员访问被拒绝。
- 权限越界请求统一返回 403。

---

### M4（2-3 天）客户端与运维收口

任务：
- 客户端新增登录态、账号切换、登出。
- 云同步页显示“当前账号 + 当前设备”。
- 部署脚本标准化：`update`, `backup`, `rollback`, `healthcheck`。
- 定时备份任务（crontab）与保留策略（例如保留 30 天）。

验收：
- 手机端脱离 USB 独立可用。
- 服务更新、回滚、备份流程可重复执行。

## 4. 数据迁移策略（不丢数据）

1. 创建默认用户（你本人）。
2. 历史数据回填 `owner_user_id=默认用户ID`。
3. 所有新写入必须携带 `owner_user_id`。
4. 迁移窗口期间限制写入（或维护模式）。
5. 迁移完成后执行全量校验与抽样核对。

回滚原则：
- 迁移前必须完整备份数据库文件。
- 任一步失败立即回退到迁移前快照。

## 5. API 改造要点

- 鉴权从全局 API Key 升级为“用户 Token + 服务 Key（可选）”双层。
- 接口保持最小集：
  - `POST /auth/login`
  - `POST /auth/logout`
  - `GET /auth/me`
  - 现有业务接口保持路径不变，增加用户隔离。
- 保持向后兼容窗口：旧客户端可读但不允许写（短期）。

## 6. 风险与控制

- 风险：SQLite 并发写锁
  - 控制：WAL + busy_timeout + 短事务 + 重任务互斥 + 重试机制
- 风险：权限遗漏导致串数据
  - 控制：仓储层统一注入 `currentUserId`，禁用“无用户上下文查询”
- 风险：迁移中断
  - 控制：迁移前快照 + 分批迁移 + 校验脚本

## 7. 执行顺序建议

1. 先完成 M0。
2. 再完成 M1（这是多人可用的准入门槛）。
3. 完成 M2 稳定性后再开放给 3-4 人试用。
4. 根据试用反馈再推进 M3、M4。

## 8. 完成定义（DoD）

- 3-4 人同时使用一周，无严重数据隔离问题。
- 关键操作成功率 >= 99%（写入、备份、恢复、列表）。
- 每日自动备份正常，且每周可演练恢复。
- 发生故障可在 30 分钟内回滚并恢复服务。
