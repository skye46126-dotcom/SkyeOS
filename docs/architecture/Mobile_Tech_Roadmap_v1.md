# LifeOS 手机端技术路线图 v1

## 阶段目标

目标：先完成长期可维护的数据底座，再逐步补交互与分析能力。

## Phase 0（已完成）

- 本地 SQLite schema v1
- Migration 执行器（V001）
- Application 启动即建库
- 数据字典 v1

验收：
- 安装启动后数据库可自动初始化
- 工程可编译打包

## Phase 1（当前进行）

- 写入模型（Project/TimeLog/Income/Expense/Learning）
- Write Repository（事务化写入）
- 用例层（Create*UseCase）
- 公共池规则：无项目绑定即 `is_public_pool=1`

验收：
- 可通过用例层写入 5 类核心数据
- 多项目绑定可写入 join 表
- 跨天记录可入库

## Phase 2

- 查询层（日报/周报/月报读取接口）
- 指标计算器（时薪、时间负债、自由度、项目时薪）
- 指标快照写入 `metric_snapshot`

验收：
- 首页不做重算，优先读快照
- 关键指标口径与数据字典一致

## Phase 3

- 备份服务（日增量/周全量/月归档）
- 恢复流程与恢复后快照重建
- 审计日志完整化

验收：
- 本地备份可恢复到指定日期
- 恢复后指标可重建且可核对

## Phase 4

- 输入体验优化（模板录入、批量录入、纠错）
- 外部解析接口（可接规则引擎或 AI）
- 待确认轻流程（非复杂队列）

验收：
- 每日录入时间控制在 5-10 分钟
- 外部解析失败不影响手动主流程

## 技术守则

- 业务逻辑不写在 Activity。
- 所有写操作必须走 UseCase -> Repository。
- 任何统计口径改动都必须同步更新数据字典。
- 新增字段必须通过 migration。
- 优先稳定性，再追求“智能化”。
