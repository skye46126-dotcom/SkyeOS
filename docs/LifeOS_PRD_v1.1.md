# LifeOS 需求文档 v1.1（可执行版）

## 1. 定位与范围

### 1.1 产品定位
LifeOS 是一个单用户、长期使用（10-30年）的人生管理系统，核心链路为：

时间 -> 项目 -> 收入 -> 自由

### 1.2 本版本范围（MVP）
本版本先实现：
- 每日自然语言输入与结构化记录
- 时间/项目/收入/支出/学习五大模块
- 日结与核心指标计算
- 日/周/月关键看板
- 本地备份

本版本暂不做强依赖项：
- 多用户协作
- 自动决策执行（仅给建议）

## 2. 已确认业务规则（来自最新确认）

- 用户类型：单用户。
- 时间记录：允许跨天（例如 23:00-01:30）。
- 项目绑定：一条记录可绑定多个项目。
- 标签：系统预置 + 用户可新增。
- 备份策略：本地备份为主。
- 成功标准：不强制 KPI 考核，但必须展示关键决策信息。

## 3. 核心数据模型

## 3.1 时间记录 TimeLog
字段：
- id
- date（记录所属自然日，跨天时按开始日期归属，同时保留结束日期）
- start_time
- end_time
- duration_hours（自动计算）
- category（工作/学习/生活/娱乐/休息/社交）
- tags（多选，可新增）
- project_ids（可多选）
- value_score（1-10）
- state_score（1-10）
- note

跨天规则：
- end_time < start_time 视为跨天，end_date 自动 +1 天。
- duration_hours 根据 start_datetime 与 end_datetime 精确计算。

## 3.2 项目 Project
字段：
- id
- name
- start_date
- status（进行中/暂停/已结束）
- ai_enable_ratio（0-100%）
- score（1-10）
- total_time_hours（自动聚合）
- total_income_amount（自动聚合）
- note

## 3.3 收入 Income
字段：
- id
- date
- source
- amount
- type（工资/项目收入/投资收入/系统收入/其他）
- project_ids（可多选，可为空）
- note

## 3.4 支出 Expense
字段：
- id
- date
- amount
- category（必要支出/体验支出/订阅/投资）
- note

## 3.5 学习 Learning
字段：
- id
- date
- content
- duration_hours
- application_level（输入学习/应用学习/结果学习）
- related_project_ids（可多选）
- tags（可多选）
- note

## 4. 未绑定项目的处理机制（重点）

“不绑定项目”并不是错误，而是归入“公共池”，用于保留真实生活与管理成本。

统一规则：
- 时间未绑定项目：进入 `公共时间池`（如通勤、管理、杂务、休闲）。
- 收入未绑定项目：进入 `公共收入池`（如工资、泛化副业收入）。
- 学习未绑定项目：进入 `公共学习池`（先学习后应用的阶段）。

分析展示规则：
- 项目看板只统计绑定到该项目的数据。
- 总览看板统计“项目池 + 公共池”。
- 系统单独展示公共池占比，帮助判断是否需要新建项目或优化归属。

## 5. 指标口径

## 5.1 核心指标
- 时薪 = 总收入 / 工作时间
- 时间负债 = 理想时薪 - 实际时薪
- 被动收入覆盖率 = 被动收入 / 生活支出
- 项目时薪 = 项目收入 / 项目投入时间
- 自由度 = 被动收入 - 必要支出

## 5.2 盈亏平衡
- 时间成本 = 投入时间 * 当前时薪基准
- 项目盈利条件：项目收入 > 时间成本

当前时薪基准（v1 固定口径）：
- 默认取“近365天实际时薪”
- 若历史不足365天，则取“已有全部历史时薪”
- 用户可手动输入“理想时薪”用于时间负债对比

## 6. AI 功能分层（v1 可落地）

- AI结构化：把自然语言转成标准记录。
- AI补全：自动建议标签/分类/项目。
- AI分析：输出周报（月报可选）。
- AI建议：给出“增加/减少时间投入”的建议项。

异常处理：
- AI 低置信度记录进入“待确认”列表。
- 用户可一键确认或快速修正。

## 7. 输入与使用流程

## 7.1 标准输入模板
```text
日期：

时间记录
- 9-12 项目A 产品设计
- 13-15 学习 LLM
- 23-01 写作复盘

收入
- 项目A 2000

支出
- 午餐 30
- 咖啡 20

学习
- Agent 工作流 2h 应用到项目A

备注
今天效率不错
```

## 7.2 每日流程（5-10分钟）
- 记录时间
- 记录收入/支出
- 记录学习（可并入时间记录）
- 1-3分钟日结

## 8. 关键展示信息（你要求“必须展示”）

首页（当天）：
- 今天时间去向（按分类 + 按项目）
- 今天收入/支出/净额
- 今天最有价值时间段
- 今天时间负债（实时）

周视图：
- 项目投入排行（时间/收入/项目时薪）
- 学习转化（输入->应用->结果）
- 公共池占比（时间/收入/学习）

月视图：
- 自由度趋势（被动收入 - 必要支出）
- 总时薪趋势 vs 理想时薪
- 项目盈亏平衡分布

## 9. 本地备份策略（主方案）

- 存储形式：本地 SQLite + 周期性导出 JSON/CSV。
- 备份节奏：
- 每日增量备份
- 每周完整备份
- 每月归档快照
- 备份目录建议：
- `backups/daily/`
- `backups/weekly/`
- `backups/monthly/`
- 恢复能力：支持按日期恢复到任意快照。
