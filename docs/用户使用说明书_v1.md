# SkyeOS（LifeOS）用户使用说明书 v1

## 1. 这是什么

SkyeOS 是一个帮助你长期管理时间、项目、收入和支出的 App。  
核心逻辑是：

`时间 -> 项目 -> 收入 -> 自由度`

你每天只需要做少量记录，系统会自动把数据汇总到“今日”和“复盘”页面。

---

## 2. 第一次使用（3 分钟）

### 第一步：进入「Settings」

填写云同步配置：

- `Server URL`：你的服务器地址（例如 `https://your-domain.com` 或 `http://你的IP`）
- `API Key`：服务器 `deploy/.env` 里的 `LIFEOS_API_KEY`
- `Device ID`：设备唯一标识（例如 `alice-android`）

点 `Save Config` 保存。

### 第二步：可选配置 AI 解析

在 `AI Parsing API` 区域填写：

- `AI Provider`：`custom / deepseek / siliconflow`
- `AI Base URL`
- `AI API Key`
- `AI Model`

点 `Test Connection (Ping)` 检查是否连通。

### 第三步：设置成本参数（建议）

- `Ideal hourly rate`（理想时薪）
- `Basic living cost`（月基础生活成本）
- `Fixed subscription cost`（月固定订阅支出）

这些参数会影响“时薪/时间负债/自由度”分析。

---

## 3. 每天怎么用（5-10 分钟）

## 3.1 在「Capture」里记录

你有两种方式：

- `Manual`：手动填表
- `AI Parse`：输入自然语言，让 AI 拆解后确认入账

手动支持 5 类记录：

- Time（时间）
- Income（收入）
- Expense（支出）
- Learning（学习）
- Project（项目）

建议每天至少记录：

1. 1-3 条时间记录  
2. 当天收入/支出  
3. 必要标签（方便后续复盘）

## 3.2 看「Today」

`Today` 页面会显示：

- Work Time（工作时长）
- Income（收入）
- Expense（支出）
- Freedom（自由度）
- Hourly Rate & Time Debt（时薪与时间负债）
- Time Allocation（时间分布）

---

## 4. 每周怎么用（10-20 分钟）

进入 `Review -> Week`，重点看：

- `Snapshot`：本周总览
- `Time Allocation`：时间分配
- `ROI Leaderboard`：高回报项目
- `Time Sinkholes`：高投入低回报项目
- `Income History`：收入历史
- `History Ledger`：历史账本
- `Tag Board`：标签维度统计

建议每周做 3 个动作：

1. 增加高 ROI 项目时间  
2. 减少 Sinkhole 时间  
3. 调整下周计划标签（如 Growth / Health / Income）

---

## 5. 主要功能说明

## 5.1 Today（今日）

- 看当天关键经营指标。
- 快速进入记录或复盘。

## 5.2 Capture（记录）

- Manual：可控、完整、精确。
- AI Parse：快输入，先预览再提交。

## 5.3 Projects（项目）

- 查看项目状态（Active / Paused / Done）。
- 在项目详情中看：
  - ROI
  - Hourly Rate
  - Time Cost / Total Cost
  - Break-even

## 5.4 Review（复盘）

- 日/周/月三种窗口分析。
- 看趋势、对比和标签效果。

## 5.5 Settings（设置）

- 云同步配置
- AI 配置
- 成本参数配置
- 备份上传、列表、下载恢复
- 标签管理

---

## 6. 备份与恢复怎么用

在 `Settings`：

1. 点 `Upload Latest Backup`：上传最新备份到服务器  
2. 点 `List Remote Backups`：查看远端备份文件名  
3. 在 `Filename` 填目标文件名  
4. 点 `Download & Restore` 执行恢复

建议：

- 每周至少上传一次备份
- 大改数据前先手动备份一次

---

## 7. 多人测试时怎么避免数据混杂

当前可用于 3-4 人测试，建议：

- 每人使用不同 `Device ID`
- App 内使用独立用户标识（如果你已启用多用户隔离方案）
- 不要共享同一设备本地数据库

## 8. 常见问题

### Q1：为什么有些指标显示 `--`？

表示该时间窗口暂无足够数据。例如你还没录入工作时长或收入。

### Q2：AI Parse 没有效果？

先检查：

1. AI Base URL / API Key / Model 是否正确  
2. `Test Connection` 是否成功  
3. 输入文本是否包含可解析信息（时间、金额、项目等）

### Q3：上传备份没反应？

先确认：

1. `Server URL` 可访问  
2. `API Key` 和服务器一致  
3. 服务器已正常运行 `lifeos-api + nginx`

## 9. 推荐使用节奏（长期可坚持）

- 每日：5-10 分钟（记录）
- 每周：10-20 分钟（复盘）
- 每月：20-40 分钟（策略调整）

持续使用后，你会越来越清楚：

- 时间花在哪
- 哪些项目值得继续
- 学习是否转化为结果
- 是否在靠近自己的“自由目标”
